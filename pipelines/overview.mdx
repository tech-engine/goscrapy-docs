Pipelines process and export the items yielded by your spider. When you call `s.Yield(record)`, the engine pushes the record into the Pipeline Manager, which passes it through all configured pipelines **sequentially in the order they appear** in `settings.go`.

## The `IPipeline` Interface

Every pipeline must implement these three methods:

```go IPipeline interface
type IPipeline[OUT any] interface {
    Open(context.Context) error
    ProcessItem(IPipelineItem, core.IOutput[OUT]) error
    Close()
}
```

| Method | Called when... | Use for... |
| :-- | :-- | :-- |
| `Open(ctx)` | Engine starts | Opening file handles, connecting to databases. |
| `ProcessItem(item, out)` | Each yielded record | Validating, transforming, or exporting data. |
| `Close()` | Engine stops | Closing files, flushing buffers, disconnecting. |

## Common Use Cases

- **Exporting** scraped data to CSV, JSON, MongoDB, Google Sheets, or Firebase.
- **Validating** records and dropping incomplete ones.
- **Deduplicating** items before export.
- **Enriching** records with additional computed fields.

> \[!IMPORTANT\] `ProcessItem` is called **concurrently** by multiple goroutines. Your pipeline implementation must be thread-safe (use `sync.Mutex` or channels if writing to shared state).
