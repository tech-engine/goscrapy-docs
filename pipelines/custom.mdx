## Scaffolding with the CLI

GoScrapy provides a built-in command to scaffold a custom pipeline:

```bash Terminal
goscrapy pipeline my_validator
```

This creates `pipelines/my_validator.go` with all three required methods stubbed out, ready for your logic.

## Generated Template

The scaffolded pipeline looks like this:

```go pipelines/my_validator.go
package pipelines

import (
    "context"
    "github.com/tech-engine/goscrapy/pkg/core"
    "github.com/tech-engine/goscrapy/pkg/engine"
)

type MyValidator[OUT any] struct {
    // add your custom fields here
}

func New[OUT any](args ...string) *MyValidator[OUT] {
    return &MyValidator[OUT]{}
}

func (p *MyValidator[OUT]) Open(ctx context.Context) error {
    return nil
}

func (p *MyValidator[OUT]) Close() {
}

func (p *MyValidator[OUT]) ProcessItem(item engine.IPipelineItem, original core.IOutput[OUT]) error {
    // original.Record()      - access the yielded Record
    // original.RecordKeys()  - access Record field names as a slice
    // original.RecordFlat()  - access Record values as []any
    // original.Job()         - access the Job
    return nil
}
```

## Dropping Items

Return `engine.ErrDropItem` from `ProcessItem` to silently drop an item. It will not be passed to subsequent pipelines.

```go pipelines/my_validator.go
func (p *MyValidator[OUT]) ProcessItem(item engine.IPipelineItem, out core.IOutput[*Record]) error {
    record := out.Record()
    if record.Title == "" {
        return engine.ErrDropItem
    }
    return nil
}
```

## Registering Your Pipeline

Add it to `settings.go`. Pipelines run in order, so place validation pipelines **before** export pipelines:

```go settings.go
var PIPELINES = []engine.IPipeline[*Record]{
    pipelines.New[*Record](), // validation first
    export2CSV,               // export only valid items
}
```

> \[!TIP\] Use `Open()` for any setup that needs the engine context (e.g., creating a file or connecting to a database). Use `Close()` to flush and release those resources.
