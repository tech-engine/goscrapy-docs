To build a custom pipeline, implement the `engine.IPipeline` interface:

```go
type IPipeline[OUT any] interface {
	Open(context.Context) error
	ProcessItem(IPipelineItem, core.IOutput[OUT]) error
	Close()
}
```

### Pipeline Methods

- **`Open(ctx)`**: Called when the engine starts. Ideal for opening files or connecting to databases.
- **`ProcessItem(item, out)`**: Called for every yielded record. Contains the core logic. **Must be thread-safe** as it is called concurrently.
- **`Close()`**: Called when the engine stops. Ideal for closing files/connections.

## Example: Validation Pipeline

If you want to explicitly drop an item (e.g., due to validation failure), return `engine.ErrDropItem` from `ProcessItem`.

```go
type ValidatePipeline struct{}

func (p *ValidatePipeline) Open(ctx context.Context) error { return nil }
func (p *ValidatePipeline) Close() {}

func (p *ValidatePipeline) ProcessItem(item engine.IPipelineItem, out core.IOutput[*Record]) error {
    record := out.Record()
    if record.Title == "" {
        // Drop the item silently
        return engine.ErrDropItem
    }
    return nil
}
```

Register it in your `settings.go`:

```go
var PIPELINES = []engine.IPipeline[*Record]{
	&ValidatePipeline{},
    export2CSV, // Will only receive valid items
}
```
