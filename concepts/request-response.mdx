## Requests (`core.Request`)

A `core.Request` represents an HTTP request to be scheduled and executed by the engine. You create one using `s.Request(ctx)` and configure it with a fluent, chainable API.

```go
req := s.Request(ctx).
    Url("https://httpbin.org/post").
    Method("POST").
    Body(`{"key":"value"}`).
    SetHeader("Content-Type", "application/json").
    Meta("category", "books").
    CookieJar("session1")
```

### Request Methods

| Method | Description |
| :-- | :-- |
| `Url(u)` | Set the target URL. Accepts a `string` or `*url.URL`. |
| `Method(m)` | Set the HTTP method (`GET`, `POST`, etc). Default: `GET`. |
| `Body(b)` | Set the request body. Accepts `string`, `[]byte`, `io.Reader`, or `io.ReadCloser`. |
| `AddHeader(k, v)` | Append a header value to a key. |
| `SetHeader(k, v)` | Set a header value, replacing any existing value for that key. |
| `Header(h)` | Replace the entire header map with `http.Header`. |
| `GetHeader(k)` | Get the value of a specific header. |
| `Meta(k, v)` | Attach metadata to pass between callbacks. |
| `CookieJar(key)` | Assign this request to a named cookie session. |
| `Context(ctx)` | Override the request context. |
| `Clone()` | Create a deep copy of this request. |

### Cookie Management

```go
req.CookieJar("session1")
```

> \[!NOTE\] If you don't set a `CookieJar` key, it defaults to `""` (empty string). This means all requests without a specific key share a **single global default cookie jar**, maintaining session state automatically across your entire spider.

## Responses (`core.IResponseReader`)

An `IResponseReader` is passed to your callback functions. It provides access to the raw response, metadata, and CSS/XPath querying.

### Response Methods

| Method | Returns | Description |
| :-- | :-- | :-- |
| `StatusCode()` | `int` | The HTTP status code (e.g., 200, 404). |
| `Body()` | `io.ReadCloser` | The raw response body stream. |
| `Bytes()` | `[]byte` | The response body as a byte slice (cached after first call). |
| `Header()` | `http.Header` | The response headers. |
| `Cookies()` | `[]*http.Cookie` | Cookies set by the server. |
| `Request()` | `*http.Request` | The original `http.Request` that generated this response. |
| `Meta(key)` | `(any, bool)` | Retrieve metadata attached to the request. |
| `Css(selector)` | `ISelector` | Query the HTML using a CSS selector. |
| `Xpath(expr)` | `ISelector` | Query the HTML using an XPath expression. |
| `Detach()` | `IResponseReader` | Create a standalone deep copy of the response (safe to use outside the callback). |

## Passing Metadata Between Callbacks

You can pass arbitrary data between callbacks using `Meta()` on the request and response.

```go
// When creating the request:
req := s.Request(ctx).Url(nextUrl).Meta("category", "books")
s.Parse(req, s.parseDetail)

// In the next callback:
func (s *Spider) parseDetail(ctx context.Context, resp core.IResponseReader) {
    category, ok := resp.Meta("category")
    if ok {
        cat := category.(string) // "books"
    }
}
```
