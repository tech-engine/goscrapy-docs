## Requests (`core.Request`)

A `core.Request` represents an HTTP request to be scheduled and executed by the engine. It provides a fluent, chainable API.

```go
req := s.Request(ctx).
    Url("https://httpbin.org/post").
    Method("POST").
    Body(`{"key":"value"}`).
    AddHeader("Content-Type", "application/json")
```

### Cookie Management

GoScrapy manages cookies seamlessly across requests using `CookieJar`.

```go
req.CookieJar("session1")
```

> \[!NOTE\] If you don't set a `CookieJar` key, it defaults to `""`. This means all requests without a specific key share a single global default cookie jar, maintaining session state automatically.

## Responses (`core.IResponseReader`)

An `IResponseReader` is passed to your callback functions. It provides access to the raw response and parsing utilities.

**Available Methods:**

- `resp.StatusCode()`: Returns the HTTP status code (int).
- `resp.Bytes()`: Returns the raw response body as a byte slice.
- `resp.Header()`: Access the `http.Header`.
- `resp.Request()`: Returns the original `core.Request` that generated this response.

## Passing Metadata

You can pass data between callbacks using `Meta()`.

```go
// In the first callback:
req.Meta("category", "books")

// In the next callback:
val, ok := resp.Meta("category")
if ok {
    category := val.(string)
}
```
