Signals are an event-driven system that lets you hook into the engine's lifecycle without modifying core logic. They decouple monitoring, metrics, and alerting from your spider code.

## Where to Wire Signals

Signals are typically wired in `base.go`, right after calling `gos.New()` and before `app.RegisterSpider(spider)`. They use a fluent builder pattern with full compile-time type safety.

```go base.go
// In base.go
func New(ctx context.Context) (*Spider, error) {
    app, _ := gos.New[*Record]()

    app.WithMiddlewares(MIDDLEWARES...).
        WithPipelines(PIPELINES...)

    spider := &Spider{ICoreSpider: app}

    // Wire signals here
    app.OnEngineStarted(func(ctx context.Context) {
        log.Println("[signal] engine started")
    }).
    OnRequestScheduled(func(ctx context.Context, req *core.Request) {
        log.Printf("[signal] request scheduled: %s", req.URL)
    }).
    OnResponseReceived(func(ctx context.Context, resp core.IResponseReader) {
        log.Printf("[signal] response: %s (status: %d)", resp.Request().URL, resp.StatusCode())
    }).
    OnItemScraped(func(ctx context.Context, item *Record) {
        log.Printf("[signal] scraped: %s", item.Title)
    }).
    OnItemDropped(func(ctx context.Context, item *Record, err error) {
        log.Printf("[signal] item dropped: %v", err)
    }).
    OnEngineStopped(func(ctx context.Context) {
        log.Println("[signal] engine stopped")
    })

    app.RegisterSpider(spider)
    go func() { _ = app.Start(ctx) }()
    return spider, nil
}
```

## Available Signals

| Category | Signal | Handler Signature |
| :-- | :-- | :-- |
| **Engine** | `OnEngineStarted` | `func(context.Context)` |
|  | `OnEngineStopped` | `func(context.Context)` |
| **Spider** | `OnSpiderOpened` | `func(context.Context)` |
|  | `OnSpiderClosed` | `func(context.Context)` |
|  | `OnSpiderIdle` | `func(context.Context)` |
|  | `OnSpiderError` | `func(context.Context, error)` |
| **Item** | `OnItemScraped` | `func(context.Context, OUT)` |
|  | `OnItemDropped` | `func(context.Context, OUT, error)` |
|  | `OnItemError` | `func(context.Context, OUT, error)` |
| **Request** | `OnRequestScheduled` | `func(context.Context, *core.Request)` |
|  | `OnRequestDropped` | `func(context.Context, *core.Request, error)` |
|  | `OnRequestError` | `func(context.Context, *core.Request, error)` |
| **Response** | `OnResponseReceived` | `func(context.Context, core.IResponseReader)` |

## Common Use Cases

- **Metrics & Monitoring**: Count requests, responses, and errors in real time.
- **Alerting**: Log or notify when items are dropped or errors spike.
- **Auto-Exit**: `spider.Wait(true)` internally uses `OnSpiderIdle` to trigger graceful shutdown.
- **Telemetry**: Feed data to the TUI stats dashboard or external monitoring systems.
