`spider.go` is where you will invest most of your time. This file contains the entire scraping logic: seeding initial requests, parsing responses, extracting data, following links, and yielding records to the pipeline.

## Triggering the Spider

There are two primary ways to kick off your scraping process.

### Option 1: Direct Call from `main.go`

This is the most common pattern. You define a `StartRequests` method on your spider and call it explicitly from `main.go`. This gives you full control over when and how the spider starts, and lets you pass in job arguments or CLI flags.

```go spider.go
// In spider.go
func (s *Spider) StartRequests(ctx context.Context) {
    s.Parse(s.Request(ctx).Url(s.baseUrl), s.parse)
}
```

```go main.go
// In main.go
spider, _ := my_project.New(ctx)
spider.StartRequests(ctx)
spider.Wait(true)
```

### Option 2: Lifecycle Hooks (Auto-Discovery)

If your spider implements any of these methods, the engine calls them automatically when registered via `app.RegisterSpider(spider)`:

| Hook | Called when... |
| :-- | :-- |
| `Open(ctx)` | Engine starts. Ideal for seeding initial requests. |
| `Close(ctx)` | Engine stops. Cleanup logic goes here. |
| `Idle(ctx)` | Spider has no active work. |
| `Error(ctx, err)` | An unhandled error occurs. |

```go spider.go
func (s *Spider) Open(ctx context.Context) {
    req := s.Request(ctx).Url(s.baseUrl)
    s.Parse(req, s.parse)
}
```

## The Parse Callback

The parse callback is where you process each response. It receives a `context.Context` and an `IResponseReader`.

```go spider.go expandable
func (s *Spider) parse(ctx context.Context, resp core.IResponseReader) {
    // 1. Log the response
    s.Logger().Infof("GET: %d %s", resp.StatusCode(), resp.Request().URL.String())

    // 2. Extract data using Gosm or selectors
    var list Listing
    _ = gosm.Map(resp, &list)

    // 3. Follow detail links
    for _, url := range list.ProductLinks {
        s.Parse(s.Request(ctx).Url(url), s.parseProduct)
    }

    // 4. Follow pagination
    if list.NextPage != "" {
        s.Parse(s.Request(ctx).Url(list.NextPage), s.parse)
    }
}

func (s *Spider) parseProduct(ctx context.Context, resp core.IResponseReader) {
    record := &Record{}
    _ = gosm.Map(resp, record)
    s.Yield(record)
}
```

<Accordion title="Key methods available on your spider">
  | Method | Description |
  | :-- | :-- |
  | `s.Request(ctx)` | Creates a new `core.Request` with a fluent API. |
  | `s.Parse(req, callback)` | Schedules a request and routes its response to a callback. |
  | `s.Yield(record)` | Sends a record to the pipeline manager for processing and export. |
  | `s.Logger()` | Returns the spider's logger instance. |
</Accordion>
