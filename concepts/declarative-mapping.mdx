Writing imperative code to manually parse deeply nested HTML elements can quickly become a painful and unmaintainable mess. **Gosm** is GoScrapy's declarative mapping engine that saves you from this headache by letting you define your data model using struct tags.

It automatically traverses nested JSON arrays, CSS, and XPath selectors to populate your records in a single line of code.

## The Power of Gosm

Whether you're dealing with deeply nested structures or complex HTML, Gosm handles the heavy lifting recursively.

```go
type Record struct {
    // Extract from JSON path 
    Title    string   `gos:"0.1.title" json:"title" csv:"title"` 
    
    // Extract from HTML using CSS or XPath
    Price    float64  `gos_css:".price_color" json:"price" csv:"price"`
    InStock  bool     `gos_xpath:"//p[@class='instock availability']" json:"in_stock" csv:"in_stock"`
    
    // Extract attributes using @attr syntax
    ImageURL string   `gos_css:"img.thumbnail@src" json:"image_url" csv:"image_url"`
    Link     string   `gos_xpath:"//a[@class='title']@href" json:"link" csv:"link"`
    
    // Recursive mapping for nested structures without imperative code
    Authors  []Author `gos:"authors" json:"authors" csv:"authors"` 
}
 
type Author struct {
    Name string `gos:"name"`
}
 
func (s *Spider) parse(ctx context.Context, resp core.IResponseReader) {
    var book Record
    // One line to map EVERYTHING
    _ = gosm.Map(resp, &book) 
    
    s.Yield(&book)
}
```
