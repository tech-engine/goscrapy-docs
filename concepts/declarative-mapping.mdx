---
title: "Declarative Mapping"
description: "Extract data automatically using Struct Tags"
---

Writing imperative code to manually parse HTML can become unmaintainable. **Gosm** is GoScrapy's declarative mapping engine that lets you define extraction rules using struct tags.

### Scenario 1: Basic HTML & Attributes

You can map CSS and XPath selectors directly to struct fields. Use `@` to target specific attributes.

```go
type Record struct {
    Title    string `gos_css:".product_main h1"`
    Price    string `gos_css:".price_color"`
    
    // Extracting an attribute
    Rating   string `gos_css:".star-rating@class"`
    NextPage string `gos_css:"li.next a@href"`
}

func (s *Spider) parse(ctx context.Context, resp core.IResponseReader) {
    var record Record
    _ = gosm.Map(resp, &record) 
}
```

### Scenario 2: JSON Extraction

If you're dealing with JSON responses, use the `gos` tag with dot-notation to navigate the JSON path.

```go
type APIResponse struct {
    ProductName string `gos:"data.0.title"`
    Price       int    `gos:"data.0.metrics.price"`
}
```

### Scenario 3: Deeply Nested Structures

Gosm handles recursion automatically. If a page has a list of items, you can map them to a slice of structs without writing imperative loops.

```go
type Page struct {
    Products []Product `gos_css:"article.product_pod"`
}

type Product struct {
    Title string `gos_css:"h3 a@title"`
    Price string `gos_css:".price_color"`
}
```
