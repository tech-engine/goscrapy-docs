---
title: "Building a custom GoScrapy middleware"
description: "Write a custom middleware by wrapping http.RoundTripper with MiddlewareFunc to modify GoScrapy requests and responses before or after the HTTP adapter."
---

A middleware in GoScrapy is a function that wraps an `http.RoundTripper`. You create them manually by implementing the wrapper pattern.

## Writing a Custom Middleware

Use `middlewaremanager.MiddlewareFunc` to wrap your logic:

```go
package middlewares

import (
    "log"
    "net/http"
    "github.com/tech-engine/goscrapy/pkg/middlewaremanager"
)

func LoggingMiddleware(next http.RoundTripper) http.RoundTripper {
    return middlewaremanager.MiddlewareFunc(func(req *http.Request) (*http.Response, error) {
        // Pre-request: modify or inspect the request
        log.Printf(">> %s %s", req.Method, req.URL.String())

        // Pass to next middleware in the chain
        resp, err := next.RoundTrip(req)

        // Post-response: inspect or modify the response
        if resp != nil {
            log.Printf("<< %d %s", resp.StatusCode, req.URL.String())
        }
        return resp, err
    })
}
```

## Registering Your Middleware

Add it to the `MIDDLEWARES` slice in your `settings.go`:

```go
var MIDDLEWARES = []middlewaremanager.Middleware{
    LoggingMiddleware,          // your custom middleware
    middlewares.Retry(),        // built-in retry
    middlewares.MultiCookieJar, // built-in cookie management
    middlewares.DupeFilter(),   // built-in duplicate filter
}
```

> \[!IMPORTANT\] Middlewares must be safe for concurrent use. Multiple goroutines will call your middleware simultaneously.
